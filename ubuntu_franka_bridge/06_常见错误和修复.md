# 常见错误和修复

## 1. libfranka 依赖错误

执行：

```bash
sudo apt --fix-broken install
sudo ldconfig
dpkg -l | grep libfranka
```

注意：

```text
ldconfig 是小写 L，不是数字 1
dpkg -l 是小写 L，不是数字 1
```

## 2. ROS 2 Humble 不存在

如果看到：

```text
/opt/ros/humble/setup.bash: 没有那个文件或目录
```

检查：

```bash
ls /opt/ros
test -f /opt/ros/humble/setup.bash && echo "ROS2 Humble OK"
```

如果没有，先运行：

```bash
./01_install_everything.sh
```

## 3. 国内 ROS 2 源不可用

检查 ROS 2 apt 源：

```bash
cat /etc/apt/sources.list.d/ros2.list
sudo apt update
```

如果清华源不可用，可以把 `ros2.list` 中的网址换成其他可访问的国内 ROS 2 镜像。

## 4. colcon build 显示 0 packages

这不是成功，说明工作空间里没有 ROS 2 包。

检查：

```bash
find ~/rdk_franka_ws/src -mindepth 2 -maxdepth 2 -name package.xml -print
```

应该看到：

```text
~/rdk_franka_ws/src/rdk_franka_bridge/package.xml
```

修复：

```bash
cd /media/wuqr/新加卷/Franka_Ubuntu_Setup
./02_build_workspace.sh
```

新版脚本会自动复制本目录中的：

```text
rdk_franka_bridge/
```

到：

```bash
~/rdk_franka_ws/src/rdk_franka_bridge
```

## 5. 启动顺序

终端 1：

```bash
./03_run_rdk_bridge.sh
```

终端 2：

```bash
source ~/.bashrc
ros2 run rdk_franka_bridge command_executor
```

终端 3：

```bash
./04_test_rdk_command.sh
```
