# 真实机械臂接入说明

`rdk_franka_bridge` 默认是安全 dry-run，不会直接让机械臂运动。

你们现在没有 `franka_ros2` 源码也没关系。当前路线改为：

```text
RDK X5 -> Ubuntu HTTP 服务 -> ROS 2 话题 -> command_executor.py -> libfranka 控制程序
```

也就是说，ROS 2 用来接收和分发指令，真正控制 Franka 可以先用 `libfranka` 直接写程序。

真正动作写在：

```bash
~/rdk_franka_ws/src/rdk_franka_bridge/rdk_franka_bridge/command_executor.py
```

主要改这几个函数：

```python
def home(self):
    ...

def stop(self):
    ...

def flip_package(self, item):
    ...

def sort_item(self, item_type, target):
    ...
```

现在已经提供了一个低速关节空间动作框架。真实动作点位优先改配置文件：

```bash
~/franka_direct_control/config/actions.conf
```

格式：

```text
点位名 q1 q2 q3 q4 q5 q6 q7 运动秒数
```

例如：

```text
home 0.000 -0.785 0.000 -2.356 0.000 1.571 0.785 5.0
```

这些关节角必须由你们现场标定，不能直接靠猜。

RDK 只负责发送高层命令：

```text
home
stop
flip_package
sort_box
sort_bag
```

Ubuntu 负责把这些命令转换为机械臂轨迹。

没有 `franka_ros2` 时，先在 `command_executor.py` 中保持 dry-run，确认 RDK 到 Ubuntu 的指令链路通了。之后再把 `home()`、`flip_package()`、`sort_item()` 这些函数改成调用你们自己的 libfranka 动作程序。

必须现场标定：

```text
home_pose
camera_pick_pose
box_left_drop_pose
bag_right_drop_pose
flip_pick_pose
flip_place_pose
```

推荐测试顺序：

```text
1. 先用 fake hardware 或仿真测试。
2. 真机只测试 home。
3. 真机只测试夹爪开合。
4. 真机低速移动到抓取点上方。
5. 真机低速抓取和放置。
6. 最后再接 RDK 自动指令。
```

RDK 发来的 HTTP 格式：

```json
{
  "command": "flip_package",
  "item": "快递件",
  "target": "翻面",
  "source": "rdk"
}
```

Ubuntu 接收地址：

```text
http://Ubuntu电脑IP:5000/robot_command
```

RDK 端桥接配置建议：

```json
{
  "mode": "http",
  "controller_url": "http://Ubuntu电脑IP:5000/robot_command",
  "timeout": 3
}
```
