# 终端命令大全

这个文件把项目里常用的终端命令集中整理。  
建议按顺序执行，不要跳步。

## 1. 进入安装文件夹

作用：进入 E 盘里的 Franka/RDK 安装包目录。

```bash
cd /media/wuqr/新加卷/Franka_Ubuntu_Setup
```

如果路径不对，先查看 E 盘在 Ubuntu 里的名字：

```bash
ls /media/wuqr
```

## 2. 给脚本添加运行权限

作用：让 `.sh` 脚本可以被终端执行。

```bash
chmod +x *.sh
```

## 3. 一键安装环境

作用：安装编译器、ROS 2、本地 libfranka、RDK bridge、C++ 控制程序。

```bash
./01_install_everything.sh
```

## 4. 单独编译 RDK bridge

作用：把 `rdk_franka_bridge` 编译成 ROS2 可以运行的包。

```bash
./02_build_workspace.sh
```

如果运行 `ros2 run rdk_franka_bridge command_executor` 提示找不到包，先执行这个。

## 5. 手动加载 ROS2 工作空间

作用：让当前终端认识 `rdk_franka_bridge` 这个 ROS2 包。

```bash
source ~/.bashrc
source ~/rdk_franka_ws/install/setup.bash
```

检查是否成功：

```bash
ros2 pkg list | grep rdk_franka_bridge
```

如果能输出：

```text
rdk_franka_bridge
```

说明 ROS2 能找到这个包。

## 6. 编译 libfranka C++ 控制程序

作用：编译我们自己写的 `franka_action_runner`。

```bash
./08_build_franka_direct_control.sh
```

编译成功后程序位置：

```bash
~/franka_direct_control/build/franka_action_runner
```

## 7. 测试 C++ 控制程序 dry-run

作用：只打印动作，不连接真实机械臂，不会让机械臂运动。

```bash
./09_test_franka_direct_dry_run.sh
```

正常会看到类似：

```text
[DRY-RUN] command=flip_package
[DRY-RUN] command=sort_box
[DRY-RUN] command=sort_bag
```

## 8. 只测试 Franka 连接

作用：只连接 Franka 控制箱并读取一次状态，不执行运动。

```bash
export FRANKA_ROBOT_IP=172.16.0.2
./10_test_franka_connection.sh
```

如果提示 `Connection timeout`，说明 Ubuntu 没有连上 Franka 控制箱，先检查网线和 IP。

## 9. 查看 Ubuntu 网络 IP

作用：查看 Ubuntu 当前所有网卡和 IP 地址。

```bash
ip addr
```

## 10. 临时设置 Ubuntu 和 RDK 通信的 IP

作用：让 Ubuntu 和 RDK 在同一网段，方便 RDK 发 HTTP 指令到 Ubuntu。

先用 `ip addr` 找到有线网卡名，例如 `enp3s0`。  
假设网卡名是 `enp3s0`：

```bash
sudo ip addr flush dev enp3s0
sudo ip addr add 192.168.127.100/24 dev enp3s0
sudo ip link set enp3s0 up
```

测试能否连接 RDK：

```bash
ping 192.168.127.10
```

## 11. 临时设置 Ubuntu 和 Franka 通信的 IP

作用：让 Ubuntu 和 Franka 控制箱在同一网段。

先用 `ip addr` 找到连接 Franka 的有线网卡名，例如 `enp3s0`。  
假设网卡名是 `enp3s0`：

```bash
sudo ip addr flush dev enp3s0
sudo ip addr add 172.16.0.1/24 dev enp3s0
sudo ip link set enp3s0 up
```

测试 Franka：

```bash
ping 172.16.0.2
```

## 12. 启动 command_server

作用：在 Ubuntu 上打开 HTTP 接收服务，等待 RDK 发指令。

终端 1 运行：

```bash
cd /media/wuqr/新加卷/Franka_Ubuntu_Setup
./03_run_rdk_bridge.sh
```

正常会看到类似：

```text

```

这个终端不要关。

## 13. 启动 command_executor

作用：监听 ROS2 指令，并调用 libfranka C++ 控制程序。

终端 2 运行：

```bash

```

正常会看到类似：

```text

```

这个终端不要关。

## 14. 本机模拟 RDK 发指令

作用：在 Ubuntu 本机假装 RDK 发一个 `flip_package` 命令，用来测试 command_server 和 command_executor 是否正常。

终端 3 运行：

```bash
cd /media/wuqr/新加卷/Franka_Ubuntu_Setup
./04_test_rdk_command.sh
```

如果成功：

- 终端 1 会显示收到 HTTP 请求。
- 终端 2 会显示收到 `flip_package`。
- 终端 2 会显示 dry-run 动作。

## 15. 查看 ROS2 话题

作用：检查 `/rdk_robot_command` 话题是否有消息。

```bash
source ~/.bashrc
source ~/rdk_franka_ws/install/setup.bash
ros2 topic echo /rdk_robot_command
```

## 16. RDK 手动发送 HTTP 测试

作用：在 RDK 上手动测试能否把命令发到 Ubuntu。

在 RDK 终端执行，假设 Ubuntu IP 是 `192.168.127.100`：

```bash
curl -X POST http://192.168.127.100:5000/robot_command \
  -H "Content-Type: application/json" \
  -d '{"command":"flip_package","item":"快递件","target":"翻面","source":"rdk_test"}'
```

如果 Ubuntu 终端收到 `flip_package`，说明 RDK 到 Ubuntu 通信成功。

## 17. RDK 端配置 Ubuntu 地址

作用：告诉 RDK，Ubuntu 的接收服务在哪里。

建议配置文件：

```bash
/root/robot_bridge/robot_bridge_config.json
```

内容示例：

```json
{
  "mode": "http",
  "controller_url": "http://192.168.127.100:5000/robot_command",
  "timeout": 3
}
```

其中 `192.168.127.100` 要换成 Ubuntu 实际 IP。

## 18. 安装开机自启动

作用：把 command_server 和 command_executor 设置为 Ubuntu 开机自动启动。

确认手动测试成功后再运行：

作用：确认自启动服务是否正在运行。

```bash

systemctl --user status rdk-command-executor.service
```

## 23. 真机执行前的环境变量

作用：让 command_executor 真正调用 libfranka 连接机械臂。

危险：没有确认安全前不要运行。

```bash
export FRANKA_ROBOT_IP=172.16.0.2
export FRANKA_EXECUTE_REAL=1
```

如果没有设置 `FRANKA_EXECUTE_REAL=1`，系统默认是 dry-run，不会真正让机械臂运动。

## 24. 修改机械臂动作点位

作用：把 RDK 的 `flip_package / sort_box / sort_bag` 转换成真实机械臂点位序列。

配置文件位置：

```bash
~/franka_direct_control/config/actions.conf
```

打开编辑：

```bash
gedit ~/franka_direct_control/config/actions.conf
```

或者用终端编辑：

```bash
nano ~/franka_direct_control/config/actions.conf
```

每一行格式：

```text
点位名 q1 q2 q3 q4 q5 q6 q7 运动秒数
```

点位名示例：

```text
home
pick_above
pick
flip_prepare
flip_place
box_left
box_right
bag_left
bag_right
```

注意：这些关节角必须由你们现场慢速标定，不要随便填。

## 26. 标定并保存一个动作点位

作用：把当前机械臂关节角保存到 `actions.conf`。

先手动引导机械臂到目标位置，例如抓取点上方。然后运行：

```bash
cd /media/wuqr/新加卷/Franka_Ubuntu_Setup
export FRANKA_ROBOT_IP=172.16.0.2
./14_calibrate_joint_pose.sh
```

按提示输入点位名，例如：

```text
pick_above
```

再输入运动时间，例如：

```text
5.0
```

确认保存后，它会自动更新：

```bash
~/franka_direct_control/config/actions.conf
```

常用点位名：

```text
home
pick_above
pick
flip_prepare
flip_place
box_left
box_right
bag_left
bag_right
```
